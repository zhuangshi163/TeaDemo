TeaWebApp

Description
Description | Vars (details) | Methods (details)

TeaWebApp is the global context that processed user's requests.

It manages the controllers in MVC pattern, handling URI requests 404 not found, redirection, etc.

This class is tightly coupled with TeaUriRouter.

		module (line 292)
	
 
	
	

Process a module from the main application.


This is similar to rerouting to a Controller. The framework offer 3 ways to process and render a module.
Based on a predefined route:
  1.  # The route is predefined in routes.conf.php
    2. 

  2.  # $route['*']['/top/:nav'] = array('MyController', 'renderTop');
    3. 

  3.  $data['top'] = Tea::app()->module('/top/banner');
    4. 


 Based on Controller name and Action method:
  1.  Tea::app()->module('MyController', 'renderTop');
    2. 

  2.  
    3. 

  3.  # If controller is in sub folder
    4. 

  4.  Tea::app()->module('folder/MyController', 'renderTop');
    5. 

  5.  
    6. 

  6.  # Passed in parameter if controller is using $this->param['var']
    7. 

  7.  Tea::app()->module('MyController', 'renderTop', array('nav'=>'banner'));
    8. 


 If class name is different from controller filename:
  1.  # filename is index.php, class name is Admin
    2. 

  2.  Tea::app()->module(array('index', 'Admin'), 'renderTop');
    3. 

  3.  
    4. 

  4.  # in a sub folder
    5. 

  5.  Tea::app()->module(array('admin/index', 'Admin'), 'renderTop');
    6. 

  6.  
    7. 

  7.  # with parameters
    8. 

  8.  Tea::app()->module(array('admin/index', 'Admin'), 'renderTop', array('nav'=>'banner'));
    9.

		throwHeader (line 423)
	
 
	
	

Analyze controller return value and send appropriate headers such as 404, 302, 301, redirect to internal routes.


It is very SEO friendly but you would need to know the basics of HTTP status code.
Automatically handles 404, include error document or redirect to inner route  to handle the error based on config ERROR_404_DOCUMENT and ERROR_404_ROUTE
Controller return value examples:
  1.  404                                  #send 404 header
    2. 

  2.  array('/internal/route', 404)        #send 404 header & redirect to an internal route
    3. 

  3.  'http://www.google.com'              #redirect to URL. default 302 Found sent
    4. 

  4.  array('http://www.google.com',301)   #redirect to URL. forced 301 Moved Permenantly sent
    5. 

  5.  array('/hello/sayhi', 'internal')    #redirect internally, 200 OK
    6.
