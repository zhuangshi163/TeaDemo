TeaValidator

Description
Description | Vars (details) | Methods (details) Constants (details)

A helper class that helps validating data.

TeaValidator can be used for form and Model data validation before saving/inserting/deleting a data.

To use TeaValidator, you have to create an instance of it and defined the validation rules. All the methods start with 'test' can be used as a rule for validating data. Rule names are case insensitive.

You can pass in custom error message along with the rules. By default all fields in the rules are required

  1.  $rules = array(
  2.               'creattime'=>array(
  3.                     array('datetime'),
  4.                     array('email'),
  5.                     array('optional')   //Optional field
  6.               ),
  7.               'username'=>array(
  8.                     array('username',6,16),
  9.                     //Custom error message will be used
  10.                     array('lowercase', 'Username must only be lowercase.')
  11.                ),
  12.  
  13.                //Only one rule
  14.                'pwd'=>array('password'),
  15.                'email'=>array('email'),
  16.                'age'=>array('between',13,200),
  17.                'today'=>array('date','mm/dd/yy'),
  18.  
  19.                //Custom rules, static method required
  20.                'a'=>array('custom', 'MainController::isA'),
  21.  
  22.                //Custom Required field message.
  23.                'content'=>array('required', 'Content is required!')
  24.         );
Rules are defined based on the validation method's parameters. For example:

  1.  //Validation method
  2.  testBetween($value, $min, $max, $msg=null)
  3.  
  4.  $rule = array(
  5.      'field'=>array('between', 0, 20)
  6.      'field2'=>array('between', 0, 20, 'Custom Err Msg!')
  7.  );
You can get the list of available validation rules by calling TeaValidator::getAvailableRules()

To validate the data, create an instance of TeaValidator and call validate() in your Controller/Model.

  1.  $v = new TeaValidator();
  2.  
  3.  # There are 3 different validation Mode.
  4.  //$v->checkMode = TeaValidator::CHECK_SKIP;
  5.  //$v->checkMode = TeaValidator::CHECK_ALL_ONE;
  6.  //$v->checkMode = TeaValidator::CHECK_ALL;
  7.  
  8.  //The $_POST or data pass in need to be an assoc array
  9.  //$data = array('username'=>'Teaphp', 'pwd'=>'12345');
  10.  if($error = $v->validate($_POST, $rules)){
  11.       print_r($error);
  12.  }
You can pass in a string to load predefined Rules which located at SITE_PATH/protected/config/forms/

  1.  <?php
  2.  //in protected/config/forms/example.php
  3.  return array(
  4.       'username'=>array(
  5.                       array('username',4,5,'username invalid'),
  6.                       array('maxlength',6,'This is too long'),
  7.                       array('minlength',6)
  8.                   ),
  9.       'pwd'=>array('password',3,5),
  10.       'email'=>array('email')
  11.   );
  12.  ?>
  13.  
  14.  //in your Controller/Model
  15.  $error = $v->validate($data, 'example');
If nothing is returned from the validate() call, it means that all the data passed the validation rules.

The Model validation rules are automatically generated when you used the framework's model generator feature. If your Models are extending TeaModel or TeaSmartModel, you can validate the data by calling TeaModel::validate()

  1.  $f = new Food;
  2.  $f->name = 'My Food name for validation';
  3.  $error = $f->validate();
  4.  
  5.  //Or...
  6.  $error = Food::_validate($f);
