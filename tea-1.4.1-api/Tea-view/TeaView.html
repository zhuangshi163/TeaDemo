TeaView

Description
Description | Vars (details) | Methods (details)

TeaView is a class for working with the "view" portion of the model-view-controller pattern.

That is, it exists to help keep the view script separate from the model and controller scripts. It provides a system of helpers, output filters, and variable escaping which is known as Template tags where you can defined them in SITE_PATH/protected/plugin/TemplateTag.php.

TeaView is a compiling template engine. It parses the HTML templates and convert them into PHP scripts which will then be included and rendered. In production mode, TeaView will not processed the template but include the compiled PHP file instead. Otherwise, it will compare the last modified time of both template and compiled file, then regenerate the compiled file if the template file is newer. Compiled files are located at SITE_PATH/protected/viewc while HTML templates should be placed at SITE_PATH/protected/view

Loops, variables, function calls with parameters, include files are supported in TeaView. TeaView only allows a template to use functions defined in TemplateTag.php. The first parameter needs to be the variable passed in from the template file and it should return a value to be printed out. Functions are case insensitive.

  1.  //register functions to be used with your template files
  2.  $template_tags = array('upper', 'toprice');
  3.  
  4.  function upper($str){
  5.      return strtoupper($str);
  6.  }
  7.  function toprice($str, $currency='$'){
  8.      return $currency . sprintf("%.2f", $str);
  9.  }
  10.  
  11.  //usage in template
  12.  Welcome, Mr. {{upper(username)}}
  13.  Your account has: {{TOPRICE(amount, 'RM')}}
Included files in template are automatically generated if it doesn't exist. To include other template files in your template, use:

  1.  <!-- include "header" -->
  2.  //Or you can use a variable in the include tag too. say, $data['file']='header'
  3.  <!-- include "{{file}}" -->
Since 1.1, If statement supported. Function used in IF can be controlled.

  1.  <!-- if -->
  2.  <!-- elseif -->
  3.  <!-- else -->
  4.  <!-- endif -->
  5.  
  6.  //Examples
  7.  <!-- if {{MyVar}}==100 -->
  8.  
  9.  //With function
  10.  <!-- if {{checkVar(MyVar)}} -->
  11.  
  12.  //&& || and other operators
  13.  <!-- if {{isGender(gender, 'female')}}==true && {{age}}>=14 -->
  14.  <!-- if {{isAdmin(user)}}==true && ({{age}}>=14 || {{age}}<54) -->
  15.  
  16.  //Write in one line
  17.  <!-- if {{isAdmin(username)}} --><h2>Success!</h2><!-- endif -->
Since 1.1, Partial caching in view:

  1.  <!-- cache('mydata', 60) -->
  2.  <ul>{{userList}}</ul>
  3.  <!-- endcache -->
TeaView can be called from the controller:

  1.  class SampleController extends TeaController{
  2.       public viewme(){
  3.           $data['myname'] = 'My name is Tea';
  4.           $this->view()->render('viewmepage', $data);
  5.           //viewmepage instead of viewmepage.html
  6.           //files in subfolder, use: render('foldername/viewmepage')
  7.       }
  8.  }
Since 1.3, you can have comments block in the template which will not be output/processed unless SHOW_TEMPLATE_COMMENT is True in common.conf.php
  1.  <!-- comment -->
  2.     this is a comment
  3.     testing {{debug(myVar)}}
  4.     <!-- include 'debugger' -->
  5.  <!-- endcomment -->
You can use native PHP as view templates in Teaphp. Use TeaView::renderc() instead of render.

In your Controller:

  1.  $data['phone'] = 012432456;
  2.  $this->view()->abc = 'ABCDE';
  3.  
  4.  //pass in true to enable access to controller if you need it.
  5.  $this->renderc('example', $data, true);
In your view scrips located in SITE_PATH/protected/viewc/

  1.  echo $this->data['phone'];     //012432456
  2.  echo $this->abc;             //ABC
  3.  
  4.  //call controller methods if enabled.
  5.  $this->thisIsFromController();
  6.  echo $this->thisIsControllerProperty;
To include a template script in viewc folder

  1.  $this->inc('this_is_another_view_php');
To write variable's value as static strings to the template, use a plus sign in front of the variable
  1.  //in controller
  2.  $this->data['siteurl'] = 'www.Teaphp.com';
  3.  $this->render('template', $this->data);
  4.  
  5.  //in template
  6.  <p>{{+siteurl}}</p>
  7.  
  8.  //The compiled template will look like:
  9.  <p>www.Teaphp.com</p>
  10.  
  11.  //Used with function call
  12.  {{+time(true)}}
  13.  
  14.  //compiled source
  15.  1262115252
Short tags with native PHP is allowed
  1.  <? echo $data; ?>
  2.  <?=$data;?>
  3.  
  4.  //The code above will be converted to
  5.  <?php echo $data; ?>
To write variable's value as static string (using native PHP), use a plus sign
  1.  //Example:
  2.  <p><?+$data;?></p>
  3.  
  4.  //result:
  5.  <p>www.Teaphp.com</p>
