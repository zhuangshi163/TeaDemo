TeaMasterSlave

TeaMasterSlave is an ORM tool based on TeaSqlMagic with Database Replication support.

This class handles Master-Slave connections. It automatically handle CRUD operations with appropriate slave/master DB server. You can handle thee connections manually by using useConnection, connectMaster, and connectSlave.

TeaMasterSlave DOES NOT send SELECT statement to a random slave. Instead, it is based on calculation with both access time and Slave nodes.

To use DB replication, you would have to setup the slave servers in db.conf.php

  1.  //This will serve as the Master
  2.  $dbconfig['dev'] = array('localhost', 'db', 'root', '1234', 'mysql',true);
  3.  
  4.  //slave with the same info as master
  5.  $dbconfig['slave'] = array('192.168.1.1', '192.168.1.2', '192.168.1.3');
  6.  
  7.  //OR ...
  8.  //slave with different info, use a string if it's same as the master info.
  9.  $dbconfig['slave'] = array(
  10.                       array('192.168.1.1', 'db', 'dairy', '668dj0', 'mysql',true),
  11.                       array('192.168.1.2', 'db', 'yuhus', 'gu34k2', 'mysql',true),
  12.                       array('192.168.1.3', 'db', 'lily', '84ju2a', 'mysql',true),
  13.                       '192.168.1.4'
  14.                    );
In the bootstrap index.php you would need to call the useDbReplicate method.

  1.  Tea::useDbReplicate();
  2.  Tea::db()->setMap($dbmap);
  3.  Tea::db()->setDb($dbconfig, $config['APP_MODE']);
