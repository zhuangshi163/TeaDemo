TeaSmartModel

Description
Description | Vars (details) | Methods (details)

TeaSmartModel is a smarter version of TeaModel that provides smart caching of the Model data.

The model classes can extend TeaSmartModel for more powerful ORM features which enable you to write shorter codes. Extending this class is optional.

All the extra ORM methods can be accessed in a static and non-static way. Example:

  1.  $food = new Food;
  2.  $food->getOne();
  3.  Food::_getOne();
  4.  
  5.  $food->count();
  6.  Food::_count();
  7.  
  8.  //Dynamic querying
  9.  $food->getById(14);
  10.  $food->getById_location(14, 'Malaysia');
  11.  
  12.  //Get only one item
  13.  $food->getById_first(14);
  14.  $food->getById_location_first(14, 'Malaysia');
  15.  
  16.  //Gets list of food with its food type
  17.  $food->relateFoodType();
  18.  $food->relateFoodType($food, array('limit'=>'first'));
  19.  $food->relateFoodType_first();
  20.  
  21.  //Static querying needs PHP 5.3 or above
  22.  Food::getById(14);
  23.  Food::getById_first(14);
  24.  Food::getById_location_first(14, 'Malaysia');
  25.  Food::relateFoodType();
  26.  Food::relateFoodType_first();
If you are using static querying methods such as Food::_count(), Food::getById(), you would need to setup the class name in your Model constructor.

  1.  Tea::loadCore('db/TeaSmartModel');
  2.  
  3.  class Model extends TeaSmartModel{
  4.      function __construct(){
  5.          parent::$className = __CLASS__;
  6.          //OR parent::setupModel(__CLASS__);
  7.      }
  8.  }
Cache are deleted automatically when Update/Insert/Delete operations occured. If you need to manually clearing the Model cache, use purgeCache()

  1.  $food->purgeCache();
  2.  //OR
  3.  Food::_purgeCache($food);
Please check the database demo MainController's test() method for some example usage.
